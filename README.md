## Abim Secondary School - Digital Portal
A modern, responsive web application for Abim Secondary School. This project features a high-performance public website and dual-role portals designed for students and parents to manage academic life and school communications.

---

## 🚀 Quick Start: How to Run This Project
If you are new to React or Vite, follow these steps to get the project running on your local machine.

1. Prerequisites
Make sure you have Node.js installed on your computer. You can download it at nodejs.org.

2. Installation
Open your terminal/command prompt, navigate to the project folder, and run:
npm install
This downloads all the necessary "ingredients" (libraries) to make the app work.

3. Launch the Localhost
Once the installation is finished, start the development server:
npm run dev

4. View the App
After running the command, the terminal will show a link, usually:
http://localhost:5173

Hold Ctrl (or Cmd) and click the link, or copy-paste it into your browser to view the site.

## 🎨 Design
Primary Colors: Navy Blue (#1e3a8a) and Ugandan Gold (#d4a855).

Typography: Bold, clean sans-serif for high readability.

Icons: Powered by Lucide-React for a modern, crisp UI.

---

## 🛠 Features
Public Website
Responsive Navigation: Fully optimized for mobile, tablet, and desktop.

School News & Events: Dynamic layouts for school updates.

Admissions & Contact: Clean, accessible information for prospective parents.

Dual Portals
Student Portal: View class schedules, track assignments, and monitor grades.

Parent Portal: Monitor child's behavior feed, check fee balances, and pay school fees online.

Bottom-Bar Navigation: Optimized mobile experience for parents on the go.

---

## ⚠️ Important Note on Portal Functionality
Please note that this version of the project is a Frontend UI/UX Prototype.

Authentication: The "Login" functionality is currently not implemented.

Access: You can access the portals directly via the "Portal" button in the navbar to explore the dashboard designs and data layouts.

Data: All data currently visible in the portals (grades, fee balances, and behavior notes) is static "mock" data for demonstration purposes.
